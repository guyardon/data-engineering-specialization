# External Integrations

**Analysis Date:** 2026-03-31

## APIs & External Services

**Notion:**
- Service: Notion Workspace (Data Engineering Specialization database)
- What it's used for: Source of truth for all course content (notes, assignments, resources)
  - SDK/Client: @notionhq/client 5.16.0
  - Integration point: `scripts/fetch-notion.mjs`
  - Auth: Environment variable `NOTION_TOKEN` (Notion API secret token)
  - Root Page ID: `139969a7aa018017b81fc3858c54fc8f`

## Data Flow

**Content Generation Pipeline:**

1. Developer runs `node scripts/fetch-notion.mjs` with `NOTION_TOKEN` set
2. Script authenticates with Notion API using token
3. Traverses Notion hierarchy: Root Page → Courses → Weeks → Sections → Subsections
4. Converts Notion blocks to Markdown/MDX:
   - Paragraphs, headings, lists, tables
   - Code blocks with syntax highlighting
   - Images (downloaded and stored locally)
   - Callouts, toggles, equations, embeds
5. Generates frontmatter metadata:
   - title, course, week, order
   - courseSlug, weekSlug for URL routing
   - notionId for potential back-references
6. Writes `.md` files to `src/content/notes/[course]/[week]/[section].md`
7. Downloads images from Notion S3 to `public/images/`
8. Astro build process:
   - Loads content via content collection API (`src/content.config.ts`)
   - Validates frontmatter against Zod schema
   - Renders MDX to static HTML
   - Outputs static files to `dist/`

## Data Storage

**Databases:**
- Primary: Notion Workspace (read-only via API)
- Local: Markdown files in `src/content/notes/` (generated)

**File Storage:**
- Images: Locally in `public/images/` (copied from Notion)
- Static assets: `public/` directory
- Built site: `dist/` directory

**Caching:**
- Generated content is gitignored: `src/content/notes/` and `public/images/` are not committed
- Must be regenerated by running `fetch-notion.mjs` after cloning

## Content Collection Schema

**Defined in:** `src/content.config.ts`

**Structure:**
```typescript
{
  title: string;              // Page title
  course: string;             // Course name (e.g., "Data Modeling")
  courseSlug: string;         // URL-safe course identifier
  courseOrder: number;        // Course sequence (1-based)
  week: string;               // Week/section name
  weekSlug: string;           // URL-safe week identifier
  weekOrder: number;          // Week sequence within course
  order: number;              // Item sequence (supports up to 3 nesting levels)
  notionId: string;           // Optional Notion page ID for back-linking
}
```

## Authentication & Identity

**Auth Provider:**
- Notion API token authentication
- Token type: Internal integration secret token
- Environment variable: `NOTION_TOKEN`
- Scope: Read-only access to Data Engineering Specialization workspace

**Note:** No user authentication in the website itself (static site). Content updates require manual script execution with valid token.

## Notion Block Types Supported

The integration script handles conversion of:
- Text blocks: paragraphs, headings (h1-h3), quotes
- Lists: bulleted, numbered, checkbox (to-do)
- Formatting: bold, italic, strikethrough, underline, code, links
- Containers: toggles, callouts, column layouts
- Media: images (downloaded locally), embeds, bookmarks
- Tables: with optional header row
- Math: equations (LaTeX)
- Dividers

## Error Handling

**Image Download:**
- Handles HTTP redirects (301/302)
- Skips images that fail to download (logs warning)
- Uses UUID extraction from Notion S3 URLs as filename

**Content Processing:**
- Escapes MDX-special characters (`<`, `>`, `{`, `}`) in text
- Handles missing page properties gracefully
- Supports pagination (max 100 blocks per request)

```

3. All subsequent API calls authenticated as integration user
```

**Page Hierarchy:**
```
Root Page (139969a7aa018017b81fc3858c54fc8f)
├── Course 1
│   ├── Week 1
│   │   ├── Section 1
│   │   │   └── Subsection 1 → generates file
│   │   └── Section 2 → generates file
│   └── Week 2 → generates file (if no subsections)
├── Course 2
└── ...
```

**Content Regeneration:**
- Script cleans `src/content/notes/` directory before running
- Downloads only images that don't already exist
- Safe to run multiple times (idempotent)
